🚀 Getting Started (5 minutes)

1

Initialize Your Client

Create a Python script to set up your federated learning client:

from realtime_fl_client import FederatedClient # Initialize client client = FederatedClient( server_url='http://127.0.0.1:5000', client_name='Your Bank Name', org_name='Your Organization' ) print("✅ Client initialized")

Parameters:

  • server_url: Central FL server address
  • client_name: Your organization's client name
  • org_name: Full organization name
2

Load Your Local Data

Load your fraud detection dataset. Your data stays completely private on your system.

import pandas as pd # Load your fraud detection data # X: Features (9 columns: Transaction_Amount, Type, Time, Device, Location, etc.) # y: Labels (0=Not Fraud, 1=Fraud) X = pd.read_csv('your_transactions.csv') # or load from database y = pd.read_csv('your_labels.csv') # Load into client (data stays private - no transmission) client.load_local_data(X, y) # Output: # 🔐 Loaded 5000 local samples (250 fraudulent, 4750 normal)
ℹ️ Privacy Note: Your transaction data is never uploaded to the server. Only model parameters are shared after local training.
3

Register with Server

Register your client with the central FL server:

# Register with server success = client.register_with_server() if success: print(f"✅ Registered. Client ID: {client.client_id}") else: print("❌ Registration failed")
✓ Success: You've been assigned a unique client ID and are ready to participate!
4

Participate in Training

Join a single training round or multiple rounds:

# Option A: Participate in ONE round client.participate_in_round() # Option B: Participate in MULTIPLE rounds (recommended) client.continuous_participation( num_rounds=10, # Total rounds to join wait_between_rounds=5 # Wait 5 seconds between rounds ) # Output: # ============================================================ # ROUND 1/10 # ============================================================ # 📥 Downloaded global model from server # 🔄 Starting local training (3 epochs)... # Epoch 1/3: Loss = 0.4523 # Epoch 2/3: Loss = 0.3891 # Epoch 3/3: Loss = 0.3421 # ✅ Local training complete. Average loss: 0.3945 # 📤 Model parameters submitted to server # ✅ Round participation complete!
5

Monitor Performance

Check client statistics and view dashboard:

# Get client statistics stats = client.get_stats() client.print_status() # Access the web dashboard # Open: http://127.0.0.1:5000/dashboard

📚 API Reference

FederatedClient Class

__init__(server_url, client_name, org_name)

Initialize a new federated learning client.

server_url: Central FL server URL (str)
client_name: Your organization's name (str)
org_name: Full organization name (str)
register_with_server() → bool

Register this client with the FL server.

Returns: True if successful, False otherwise
load_local_data(X, y) → bool

Load local fraud detection data (stays private).

X: Features dataframe (pd.DataFrame)
y: Labels series (pd.Series)
Returns: True if successful
participate_in_round() → bool

Participate in a single federated training round.

Returns: True if successful
continuous_participation(num_rounds=10, wait_between_rounds=5) → void

Participate in multiple training rounds.

num_rounds: Number of rounds (int)
wait_between_rounds: Wait time in seconds (int)
get_stats() → dict

Get current client statistics.

Returns: Dictionary with client stats

🔧 Troubleshooting

Connection Issues

Problem: "Connection error: [Errno 111] Connection refused"

  • Ensure the FL server is running: python flask_app_realtime.py
  • Check server URL matches your setup (default: http://127.0.0.1:5000)
  • Verify no firewall blocking the connection

Data Format Issues

Problem: "IndexError: invalid index"

  • Ensure X has exactly 9 numerical features
  • Ensure y has binary values (0 or 1)
  • Check for missing values: X.isnull().sum()

No Data Loaded

Problem: "No data loaded. Call load_local_data() first"

  • Call load_local_data(X, y) before participate_in_round()
  • Verify dataset is not empty: len(X) > 0

💡 Best Practices

Data Preparation

  • ✓ Normalize/scale numerical features before loading
  • ✓ Encode categorical features as integers
  • ✓ Remove any personally identifiable information (PII)
  • ✓ Handle missing values appropriately
  • ✓ Ensure consistent feature order across all uploads

Participation Strategy

  • ✓ Start with sample data to test integration
  • ✓ Participate in all training rounds for maximum benefit
  • ✓ Monitor dashboard performance after each round
  • ✓ Provide diverse data (different regions, times, transaction types)
  • ✓ Ensure reliable network connection during participation

Privacy & Security

  • ✓ Use HTTPS in production (not HTTP)
  • ✓ Keep your client credentials private
  • ✓ Validate server certificates before connecting
  • ✓ Regularly update the client library
  • ✓ Never share your client_id publicly

❓ FAQ

Q: Will my data be shared with other organizations?

A: No. Your raw data stays on your system. Only trained model parameters are shared.

Q: How much data do I need?

A: Minimum 1000 samples recommended. More data improves model contributions.

Q: Can I participate part-time?

A: Yes! Participate in as many rounds as you want. Each round is independent.

Q: What if our model doesn't match others?

A: FedAvg (Federated Averaging) automatically handles heterogeneous models. All local and global models have same architecture.

Q: How is model quality ensured?

A: The server evaluates the aggregated model each round. Poor qualityupdates are downweighted by data size.

Q: Can we withdraw?

A: Yes, stop running the client and inform administrators. Your data remains private.
